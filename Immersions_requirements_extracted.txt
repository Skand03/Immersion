Immersions@WPU
ReactJs Assessment
Implement a Destination Flow
Your task is to build a web interface in react that allows a user to add a new rural
immersion destination to the GAIA system. You need to create a functional component
with a form, manage any client-side validations while ensuring server-side validations.
The interface should gracefully handle any exception (client and server side). Every API
will raise an exception with a status code and a detail.
Be creative. The assignment will be judged on the basis of code quality, user experience,
and overall functionality. Use of AI is not prohibited. Please make sure to show your
preferred AI agent’s chat for brownie points. You have to implement 4 APIs.
All APIs need a mandatory header
Authorization: Bearer {token}
The API-domain and a test token will be provided to you.
All the best!

-- 1 of 3 --

API 1 – CREATE DESTINATION
POST
{domain}/admin/create-destination
Request Body:
Field Type Required Description
name String YES Destination name
description String YES Detailed description
meta String Optional Metadata/tags
On Success Response:
{
"statusCode": 201,
"data": {
"destinationId": 1
}
}
API 2 – UPDATE DESTINATION
PUT
{domain}/admin/update-destination/{destinationId}
Request Body:
Field Type Required Description
name String OPTIONAL Destination name
description String OPTIONAL Detailed description
On Success Response:
{
"statusCode": 201,
"data": {
"destinationId": 1
}
}

-- 2 of 3 --

API 3 – DELETE DESTINATION
DELETE
{domain}/admin/delete-destination/{destinationId}
Request Body: None
On Success Response: status code 204
API 4 – GET ALL DESTINATIONS
GET
{domain}/admin/destinations
Query Parameters:
Parameter Type Default Purpose
page Integer 1 Page number
perPage Integer 10 Records per page
name String — Search filter
On Success Response:
{
"statusCode": 200,
"data": {
"destinations": [

"destinationName": "Ralegan Siddhi",

"createdBy": "John Doe"
}
],
"metadata": {
"total": 45,

"perPage": 10
}
}
}
